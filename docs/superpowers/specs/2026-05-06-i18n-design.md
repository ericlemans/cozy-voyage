# i18n: German / English — Design Spec

**Date:** 2026-05-06  
**Status:** Approved

---

## Goal

Add German/English language switching to the Cozy Voyage website. Language is reflected in the URL (`/de/...`, `/en/...`). Browser language auto-selects the locale on first visit. A toggle in the header lets users switch at any time.

---

## Library

**`next-intl`** — purpose-built for Next.js App Router. Provides:
- Middleware-based locale routing and detection
- `useTranslations(namespace)` hook for components
- `getTranslations(namespace)` for server components / pages
- Type-safe keys via generated types

Install: `npm install next-intl`

---

## URL Structure

| Locale | URL prefix | Example |
|--------|-----------|---------|
| German (default) | `/de` | `/de`, `/de/impressum` |
| English | `/en` | `/en`, `/en/impressum` |

Visiting `/` redirects to `/de` or `/en` based on `Accept-Language` header, falling back to `de`.

---

## App Router Restructure

```
app/
  [locale]/
    layout.tsx        ← wraps NextIntlClientProvider, sets <html lang>
    page.tsx          ← home page
    agb/page.tsx
    impressum/page.tsx
    privacy-policy/page.tsx
  globals.scss        ← stays at root (not locale-specific)
```

`app/layout.tsx` is removed; `app/[locale]/layout.tsx` becomes the root layout. It receives the `locale` param, loads the matching messages JSON, and wraps children in `NextIntlClientProvider`.

---

## Middleware

`middleware.ts` at the project root:
- Detects locale from `Accept-Language` header
- Redirects `/` → `/de` or `/en`
- Passes locale to all matched routes
- Matcher: `['/((?!_next|assets|favicon.ico).*)']`

## Components to Translate

All UI text is extracted. Data that is proper nouns (city names, guest names, property titles) stays hardcoded in German as it appears on the booking platform.

| Component | Strings to extract |
|-----------|-------------------|
| `HomeBanner` | Eyebrow, subheadline, feature tags, CTA button, destination link labels |
| `Properties` | Label, heading, subtitle, card labels (guests, price, "ab"), CTA text |
| `CompanyStandards` | Label, heading, subtitle, all 6 feature titles + descriptions |
| `Gallery` | Label, heading, subtitle, scroll button aria-labels |
| `TestimonialCarousel` | Label, heading, subtitle, testimonial quotes + guest listings |
| `Footer` | Tagline, nav headings, link labels, copyright |
| `Header` | Language toggle label (DE / EN) |

### Legal pages (agb, impressum, privacy-policy)

These pages are legally binding in German. The legal body text stays in German. Only the page `<title>` and any surrounding navigation/UI text is translated.

---

## Language Switcher

A simple `DE / EN` text toggle added to the Header (which is currently empty/commented out). On click it navigates to the same pathname with the other locale prefix using `useRouter` + `usePathname` from `next-intl/navigation`.

Position: top-right of the header, consistent with the site's minimal aesthetic. Style: small uppercase text, rose accent on the active locale, `/` separator.

---

## Type Safety

After installing next-intl, run the type generation step so `useTranslations` keys are validated at compile time. Add `next-intl/plugin` to `next.config.ts`.

---

## Out of Scope

- More than 2 languages
- Translated URLs (e.g. `/en/impressum` stays as-is, not `/en/legal-notice`)
- Any backend / booking widget translation (Lodgify handles its own locale)
